# 📞 Complete Call Request Flow - MeDora TeleMedicine

## Overview
This document explains the complete flow when a patient initiates a video call request to a doctor.

---

## 🔄 Call Flow Sequence

### Step 1: Patient Initiates Call Request

**Location:** `PatientDashboard.tsx`

```typescript
const startConsultation = async (doctorId: string) => {
  // 1. Create call record in backend
  const response = await callAPI.initiate({
    initiatorId: user?.id,
    receiverId: doctorId,
    callType: 'VIDEO'
  });
  
  const callData = response.data.call;
  const roomId = callData.roomId; // UUID generated by backend
  
  // 2. Send consultation request via WebSocket
  socketService.emit('consultation_request', {
    callId: callData.id,
    patientId: user?.id,
    patientName: user?.name,
    doctorId: doctorId,
    roomId: roomId,
    reason: 'Video consultation'
  });
  
  // 3. Show waiting message
  toast.info('Call request sent to doctor. Waiting for response...');
};
```

**What happens:**
- Call record created in database with status `RINGING`
- Unique `roomId` generated (UUID)
- WebSocket event sent to call server
- Patient sees "Waiting for response..." message

---

### Step 2: Call Server Routes Request to Doctor

**Location:** `call-server/server.js`

```javascript
socket.on('consultation_request', (data) => {
  const doctor = onlineDoctors.get(data.doctorId.toString());
  
  if (doctor) {
    // Send to specific doctor's socket
    io.to(doctor.socketId).emit('consultation_request', {
      callId: data.callId,
      patientId: data.patientId,
      patientName: data.patientName,
      reason: data.reason,
      roomId: data.roomId,
      timestamp: Date.now()
    });
  } else {
    // Doctor not online
    socket.emit('consultation_error', {
      message: 'Doctor is not available'
    });
  }
});
```

**What happens:**
- Server checks if doctor is online
- If online: Routes request to doctor's specific socket connection
- If offline: Sends error back to patient

### Step 4A: Doctor Accepts Call

**Location:** `DoctorDashboard.tsx`

```typescript
const acceptCall = async () => {
  // 1. Update call status in backend
  await callAPI.acceptCall(incomingCall.callId);
  
  // 2. Notify patient via WebSocket
  socketService.emit('consultation_accepted', {
    callId: incomingCall.callId,
    patientId: incomingCall.patientId,
    doctorId: user?.id,
    doctorName: user?.name,
    roomId: incomingCall.roomId
  });
  
  // 3. Navigate to video call
  navigate(`/video-call?room=${incomingCall.roomId}`);
};
```

**Backend:** `CallController.java`
```java
@PutMapping("/{id}/accept")
public ResponseEntity<?> acceptCall(@PathVariable Long id) {
    Call call = callService.acceptCall(id);
    return ResponseEntity.ok(Map.of("call", call));
}
```

**What happens:**
- Call status updated to `ACCEPTED` in database
- WebSocket event sent to patient
- Doctor navigates to video call page

### Step 4B: Doctor Rejects Call

**Location:** `DoctorDashboard.tsx`

```typescript
const rejectCall = async () => {
  // 1. Update call status in backend
  await callAPI.rejectCall(incomingCall.callId);
  
  // 2. Notify patient via WebSocket
  socketService.emit('consultation_rejected', {
    callId: incomingCall.callId,
    patientId: incomingCall.patientId,
    reason: 'Doctor is busy'
  });
  
  toast.info('Call declined');
};
```

**What happens:**
- Call status updated to `REJECTED` in database
- WebSocket event sent to patient
- Patient sees rejection message

---

### Step 5: Patient Receives Response

**Location:** `PatientDashboard.tsx`

```typescript
// Listen for acceptance
socketService.on('consultation_accepted', (data: any) => {
  if (data.callId === callData.id && data.patientId === user?.id) {
    toast.success(`Dr. ${data.doctorName} accepted your call!`);
    navigate(`/video-call?room=${data.roomId}`);
  }
});

// Listen for rejection
socketService.on('consultation_rejected', (data: any) => {
  if (data.callId === callData.id && data.patientId === user?.id) {
    toast.error(data.reason || 'Doctor declined the call');
  }
});
```

**What happens:**
- If accepted: Patient navigates to video call page
- If rejected: Patient sees error message

### Step 6: Both Join Video Call Room

**Location:** `VideoCallPage.tsx`

```typescript
useEffect(() => {
  const roomId = searchParams.get('room');
  
  // Get media devices
  navigator.mediaDevices.getUserMedia({ video: true, audio: true })
    .then(stream => {
      setLocalStream(stream);
      
      // Join room via WebSocket
      socket.emit('join-room', {
        roomId,
        userId: user.id,
        userName: user.name,
        userType: user.role.toLowerCase()
      });
      
      // Setup WebRTC peer connection
      setupPeerConnection(stream);
    });
}, []);
```

**What happens:**
- Both users access camera/microphone
- Both join same WebSocket room
- WebRTC peer connection established
- Video/audio streams exchanged

## 🗄️ Database Schema

### Call Table
```sql
CREATE TABLE calls (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  initiator_id BIGINT NOT NULL,
  receiver_id BIGINT NOT NULL,
  call_type VARCHAR(20) NOT NULL,
  status VARCHAR(20) NOT NULL,
  room_id VARCHAR(255) NOT NULL,
  started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  ended_at TIMESTAMP NULL,
  duration BIGINT NULL
);
```

### Call Status Flow
```
RINGING → ACCEPTED → ENDED
        ↘ REJECTED
```

## 🔌 WebSocket Events

### Patient → Server
- `consultation_request` - Initiate call

### Server → Doctor
- `consultation_request` - Notify doctor of incoming call

### Doctor → Server
- `consultation_accepted` - Accept call
- `consultation_rejected` - Reject call

### Server → Patient
- `consultation_accepted` - Doctor accepted
- `consultation_rejected` - Doctor rejected
- `consultation_error` - Doctor not available

### Both → Server (Video Call)
- `join-room` - Join video room
- `offer` - WebRTC offer
- `answer` - WebRTC answer
- `ice-candidate` - ICE candidate
- `end-call` - End call

## 🐛 Troubleshooting

### Issue: Doctor not receiving notification
**Check:**
- Doctor's online status in database
- WebSocket connection active
- Call server logs

### Issue: Patient not navigating to video call
**Check:**
- WebSocket event listeners registered
- Room ID matches
- Navigation permissions

### Issue: Video call not connecting
**Check:**
- Camera/microphone permissions
- WebRTC peer connection
- STUN server accessibility
